# План 1: Устранение дублирования данных и модульность кода

## Цель
Сократить дублирование кода, создать единый источник данных и упростить поддержку проекта.

## Шаги реализации

### 1. Создать единый файл данных
- Объединить `products-data.json` и `brands-data.json` в `catalog-data.json`
- Добавить связь между брендами и продуктами через поле `brandId` в продуктах

### 2. Реализовать модульную структуру
- Создать папку `js/modules/` для общих функций
- Вынести повторяющиеся функции в отдельные модули:
  - `data-loader.js` - загрузка и управление данными
  - `components-loader.js` - загрузка хедера, футера, сайдбара
  - `ui-utils.js` - утилиты для работы с UI

### 3. Обновить существующие скрипты
- Переработать `product-loader.js` для использования модулей
- Переработать `brands-loader.js` для использования модулей
- Переработать `script.js` для использования модулей

### 4. Проверить работу сайта
- Запустить сайт локально
- Проверить работу всех страниц
- Убедиться, что данные загружаются и отображаются корректно

## Результат
- Единый источник данных для всех страниц
- Модульный код, удобный для поддержки
- Удаление дублирования функций
