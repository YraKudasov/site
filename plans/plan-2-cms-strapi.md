# План 2: Переход на CMS Strapi

## Цель
Внедрить управление контентом через CMS для удобного редактирования данных без прямого доступа к коду.

## Шаги реализации

### 1. Установка и конфигурация Strapi
- Установить Node.js и npm
- Создать проект Strapi
- Конфигурация базы данных (SQLite для локального разработки)

### 2. Создание контентных типов
- Создать Content Type "Brand" (Бренд)
  - Поля: id, name, title, description
- Создать Content Type "Product" (Продукт)
  - Поля: id, name, title, description, image, specs, ctaTitle, isActive, brand (relation to Brand)

### 3. Импорт данных
- Импортировать существующие данные из JSON-файлов в Strapi
- Проверить правильность связей между брендами и продуктами

### 4. Настройка доступа к API
- Сконфигурировать публичный доступ к API
- Настроить CORS для разрешенных доменов

### 5. Обновление фронтенда
- Изменить `data-loader.js` для запросов к Strapi API вместо локальных JSON-файлов
- Обновить URL-адреса запросов
- Добавить обработку ошибок для API-запросов

### 6. Развертывание
- Развернуть Strapi на хостинге (Render, Heroku, VPS)
- Настроить постоянное хранилище для медиафайлов (AWS S3, Cloudinary)

## Результат
- Панель управления контентом для редактирования данных
- REST API для доступа к данным
- Возможность добавления/удаления продуктов и брендов без изменения кода
- Автоматическая генерация схемы API с документацией
